> ⚠️ 2026-06-10 更新：开盘啦API实测失效已退役（脚本归档于 `archive/kailai_api.py`），涨停股数据已迁至 AkShare `stock_zt_pool_em`；原第6/7数据源（开盘啦多日叠加/异动提醒）已从代码中删除，实际为5源。`integrated_review.py`（V4.0整合脚本）已归档至 `archive/`。

## 📋 系统概述

多维度复盘系统整合了5个数据源和8维度评分体系，帮助你全面筛选优质标的，避免遗漏机会。

## 🎯 核心功能

### 1. 5源股票池构建
- ✅ 资金流入TOP20（AkShare）
- ✅ 人气股TOP10（AkShare - 按成交额）
- ✅ 所有涨停股（AkShare `stock_zt_pool_em`，原开盘啦API已失效）
- ✅ 龙虎榜股（AkShare）
- ✅ 强度前5板块的前5个股（AkShare）
- ❌ 开盘啦多日叠加（已随开盘啦接口失效删除）
- ❌ 开盘啦异动提醒（已随开盘啦接口失效删除）

### 2. 8维度量化评分（总分13分）
| 维度 | 满分 | 说明 |
|------|------|------|
| 是否有逻辑 | 3分 | 政策/涨价/业绩=3分，AI/新能源=2分 |
| 走的主动的 | 2分 | 早盘涨停=2分，上午=1分，下午=0分 |
| 带动板块的 | 1分 | 板块最早涨停=1分 |
| 是否是龙1 | 2分 | 3板+板块第一=2分，板块前2=1分 |
| 有跟风股 | 1分 | 出现在3个以上数据源=1分 |
| 未开始加速 | 1分 | 1-2板=1分，3-5板=0.5分，6板+=0分 |
| 转弱转强 | 2分 | 2板+高换手=1分（反包） |
| 有点容量 | 1分 | 成交额>10亿=1分 |

**评分标准**：
- 🔥 >=9分：重点关注
- ⚠️ 6-8分：观察
- ❌ <6分：不关注

## 🚀 使用方法

### 方法1：独立运行多维度分析

```bash
# 分析今天
~/.claude/mcp-servers/astock-data/venv/bin/python3 ~/.claude/skills/fupan/multi_dimension_review.py

# 分析指定日期（仅龙虎榜/涨停池支持历史，其余为实时数据，需 --force）
~/.claude/mcp-servers/astock-data/venv/bin/python3 ~/.claude/skills/fupan/multi_dimension_review.py --date 2026-04-30 --force
```

**输出内容**：
1. 股票池构建过程（5个数据源）
2. 8维度评分详情
3. 重点关注股票列表（>=9分）
4. 观察股票列表（6-8分）
5. 统计信息（来源分布）
6. JSON结果文件（保存到 `/tmp/fupan_multi_dimension_YYYYMMDD.json`）

### 方法2：在 /fupan 中使用

在运行 `/fupan` 时，Claude会自动调用多维度分析工具，整合到完整复盘流程中。

**完整流程**：
1. 多维度股票池构建 + 8维度评分
2. V4.0框架周期判断（首波/补涨/二波龙）
3. 情绪温度计算 + 仓位管理

## 🔧 数据源说明

### 已实现的数据源

1. **涨停股数据**（AkShare，原开盘啦API已于2026-06失效）
   - 接口：`ak.stock_zt_pool_em(date="YYYYMMDD")`
   - 包含：代码、名称、涨跌幅、连板数、封板资金、首次封板时间、炸板次数、换手率、成交额、所属行业
   - 注意：所属行业≠题材概念，主线归类走 plate_classifier

2. **资金流入数据**（AkShare）
   - 接口：`ak.stock_individual_fund_flow_rank(indicator="今日")`
   - 包含：主力净流入金额、净流入占比
   - 注意：接口不稳定，可能失败

3. **人气股数据**（AkShare）
   - 接口：`ak.stock_zh_a_spot_em()`
   - 按成交额排序取TOP10
   - 注意：接口不稳定，可能失败

4. **龙虎榜数据**（AkShare）
   - 接口：`ak.stock_lhb_detail_em()`
   - 包含：上榜原因、龙虎榜净买额、涨跌幅
   - 稳定性：较好

5. **板块龙头数据**（AkShare）
   - 接口：`ak.stock_board_industry_name_em()` + `ak.stock_board_industry_cons_em()`
   - 取强度前5板块的前5个股
   - 注意：接口不稳定，可能失败

### 已删除的数据源（开盘啦接口2026-06实测失效）

6. **开盘啦多日叠加**（已删除）
   - AkShare 近似替代：涨停池"涨停统计"字段（N天M板）+ `ak.stock_zt_pool_strong_em()`

7. **开盘啦异动提醒**（已删除）
   - AkShare 替代：`ak.stock_changes_em()` 盘口异动（盘中监控用，复盘时价值较低）

## ⚠️ 注意事项

### 数据获取失败处理
- AkShare接口不稳定，可能因为网络或东方财富服务器问题失败
- 如果某个数据源失败，系统会继续使用其他数据源
- 至少保证涨停股数据（AkShare `stock_zt_pool_em`）和龙虎榜数据（AkShare）可用

### 评分逻辑的局限性
1. **逻辑判断**：基于概念关键词匹配，可能不够精准
2. **跟风股判断**：暂时用数据源数量代替，不够准确
3. **反包判断**：需要历史K线数据，目前只能简单判断
4. **大盘区间强度**：需要分时数据，暂未实现

### 建议
- 多维度评分是辅助工具，不能完全替代人工判断
- 重点关注（>=9分）的股票仍需结合V4.0框架的周期判断
- 最终决策要考虑情绪温度、仓位管理等因素

## 🆘 常见问题

### Q1: 为什么有些数据源获取失败？
A: AkShare接口依赖东方财富服务器，可能因为网络或服务器问题失败。建议多试几次，或者等待服务器恢复。

### Q2: 评分结果准确吗？
A: 评分是基于量化指标的辅助工具，不能完全替代人工判断。建议结合V4.0框架的周期判断和自己的经验。

### Q3: 如何调整评分权重？
A: 可以修改 `multi_dimension_review.py` 中的评分逻辑。比如牛市提高"容量"权重，熊市提高"逻辑"权重。

### Q4: 能否增加自定义数据源？
A: 可以。在 `MultiDimensionReview` 类中添加新的 `_get_xxx()` 方法，然后在 `get_all_stocks_to_review()` 中调用。
